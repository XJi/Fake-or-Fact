OpenSources.co Analysis

We trivially compare the article URL with the curated list of news sites maintained by OpenSources.co. This provides a good benchmark on how a human maintained list performs against the large amount of news sites on the internet. The number one challenge with having humans classify news articles as credible and non-credible is that humans are prone to judgement errors and that the pace of news articles often surpasses the speed at which a person can possibly review them.

Headline Extraction - Machine Learning Module

Our application attempts to extract the headline from a online news article. Next, we extract features from the headline and test it against our training set to make a judgement about wheter or not the news article is credible or non-credible. Scroll down below to see how our individual machine learning models performed!

Content Extraction - Machine Learning

Finally, our application extracts the content of a news articles and determines the article keywords based on frequency of usage. The keywords are used as features in different machine learning models. Once again, feel free to scroll down and see how each model performed.

SVM Prediction:

{{ entries[7] }}

Support Vector Machine (SVM) is a powerful linear classification algorithm. By solving the maximum margin problem, it aims at finding a hyperplane decision boundary in the middle of the two classes. In our model, the soft-margin version is used.

Decision Tree Prediction:

{{ entries[5] }}

Decision tree is a tree model, the query process corresponds to a path from the root to a leaf. In each inner node one feature value will be examined, by comparing with a pre-calculated value it will decide which subtree to go. When it reach a leaf, the result stored in will be the answer. In our project, Gini impurity is used to determine which feature to examine for every node when building the tree.

Random Forest (decision tree version) Prediction:

{{ entries[4] }}

The random forest is a collection of different trees which are trained by different sub-samples of the dataset, therefore random forest is less susceptible to over-fitting when compared with many other models. This version is constructed base on decision trees which use Gini impurity when training.

Random Forest (random tree version) Prediction:

{{ entries[3] }}

This is also a random forest, what's different is the trees inside are random trees, they are not selecting features based on measurements like Entropy and Gini impurity, but by randomly choosing instead. The method is proposed in a paper by Adele Cutler and Guohua Zhao, "PERT – Perfect Random Tree Ensembles".

Neural Network Prediction:

{{ entries[6] }}

The neural network working behind is a feed-forward neural network. It contains 105 neurons(perceptrons), divided into 3 layers. Neurons in the 1st layer will read in the feature, and neurons in the remaining 2 layers will read output of neurons from last layer.

LSTM Prediction:

{{ entries[9] }}

LSTM refers to Long-Short Term Memory, which is a recurrent neural network architecture. It achieved the best known results in natural language text. This model applies binary cross－entropy to the preprocessed content keywords data.
